# 基于深度学习的信用风险评估框架 (Deep Learning in Credit Scoring)

## 项目简介
信用风险评估对于维护金融系统的稳定运作至关重要。在如今信息爆炸的时代，传统机器学习方法在处理大规模、高维且非线性数据时面临能力不足的问题。本项目提出了一套兼顾**准确性、公平性、透明度和隐私性**的高可靠性信用风险评估解决方案。

本项目引入深度学习进入信用风险评估框架，以专为表格数据设计的 **TabTransformer** 模型为核心，系统性地应对了数据不平衡、公平性、模型透明度以及数据隐私等多重挑战，为满足现实金融监管要求提供了新的理论和实践支持。

## 关键词

信用评估 | 深度学习 | 不平衡性 | 模型可解释性 | 公平性 | 联邦学习 | TabTransformer | SHAP | FedAvg

## 核心特性
- **核心分类模型**：采用专为表格数据设计的深度学习模型 **TabTransformer**，强大的特征提取能力使其能够完美适配结构化的信用数据。
- **数据不平衡处理**：采用 **SMOTEC** 过采样技术，有效应对信用评估中常见的违约/守约样本极度不平衡问题。
- **模型公平性约束**：借鉴 **Prejudice Remover** 思想，在损失函数中引入基于分离性标准的正则项，有效消除模型对特定群体的偏见。
- **透明化可解释性 (XAI)**：应用适用于分类型数据和连续型数据的**定制化 SHAP 方法**，打破深度学习的“黑匣子”，实现模型预测的透明化解释。
- **数据隐私保护**：采用联邦学习方法 **FedAvg**，实现多方机构在保护各自数据隐私的前提下，联合利用多方数据训练全局模型。

## 技术贡献
- 当前SHAP的Python库并不能直接适配同时具有连续型变量和分类型变量的情况，因此在他们的思想基础上均进行了调整以适应这种多类型数据的情况。

## 项目结构
结合本仓库的代码实现，主要结构如下：
- **`code.ipynb`**：核心代码库（Jupyter Notebook）。包含了数据预处理、SMOTEC 过采样实现、TabTransformer 模型训练、模型公平性正则项的添加，以及定制化 SHAP 解释性分析的代码实现。
- **`federal/`**：联邦学习模块。包含使用 FedAvg 算法实现多方数据联合训练的代码与实验配置。
- **`Default of Credit Card Clients Dataset （台湾）30000行/`**：台湾信用卡违约数据集。包含了 30,000 条台湾信用卡客户的真实借款和还款记录。
- **`German/`**：德国信用数据集。包含了用于评估客户信用风险的分类与连续型特征数据。
- **`statlog australian credit approval 690行/`**：澳大利亚信用审批数据集。包含了 690 条信用卡的申请记录信息。
- 注：由于阿里天池比赛数据集的数据集过大，无法直接上传，因此暂无提供

## 实验数据集
本项目在以下四个真实信用风险数据集上进行了严格的实验验证：
1. 澳大利亚信用数据集 (Australian)
2. 德国信用数据集 (German)
3. 台湾信用卡违约数据集 (Taiwan)
4. 阿里天池比赛数据集 (Tianchi)

## 实验结果
通过在上述四个真实数据集上的实验验证，得出以下主要结论：
1. 经过不平衡性处理后，TabTransformer 模型在 AUC 指标上表现稳健且优越，**显著优于传统 MLP，并能与 XGBoost 相媲美**。
2. 所引入的公平性约束和联邦学习机制均能在有效解决算法偏见和隐私保护问题的同时，将对模型预测性能的负面影响降至最低。
3. 验证了 TabTransformer 及深度学习生态在信用风险评估领域的巨大潜力和适用性。

```bash

cd Deep-Learning-in-Credit-Scoring

```

### 2. 运行核心代码

建议在本地或云端的 Jupyter 环境中运行。
打开 `code.ipynb` 并按顺序运行单元格，您可以重现数据处理、模型训练、模型公平化约束以及模型透明性解释（SHAP）的全过程。

进入 `federal/` 目录，根据该目录下的具体脚本文件执行联邦学习 (FedAvg) 相关的模型训练与评估流程。
